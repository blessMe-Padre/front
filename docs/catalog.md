# Каталог и фильтры

Фильтры каталога работают через query-параметры URL. Страница `app/(pages)/catalog/[slug]/page.tsx` читает `searchParams`, нормализует их через `getCatalogFilters`, загружает товары и передает состояние фильтров в `CatalogFilters`.

## Основные файлы

- `app/(pages)/catalog/[slug]/page.tsx` загружает категорию, товары и варианты фильтров.
- `app/(pages)/catalog/components/CatalogFilters.tsx` отвечает за интерфейс фильтров.
- `app/utils/catalogQueries.ts` парсит query-параметры и собирает Strapi URL для товаров.
- `app/types/types.ts` описывает тип `CatalogSearchParams`.

## Query-параметры

Фильтры сохраняются в URL:

```text
/catalog/epoksidnaya-smola?category=nalivnoy-pol&priceFrom=1000&priceTo=5000&inStock=1&manufacturer=A&manufacturer=B
```

Поддерживаемые параметры:

- `category` — slug выбранной подкатегории.
- `priceFrom` — минимальная цена.
- `priceTo` — максимальная цена.
- `inStock=1` — только товары в наличии.
- `manufacturer` — выбранные производители, параметр может повторяться.
- `country` — выбранные страны происхождения, параметр может повторяться.
- `uvResistance` — выбранные значения УФ-стойкости, параметр может повторяться.

## Серверная фильтрация Strapi

Все товарные фильтры применяются в Strapi-запросе до пагинации результата:

```ts
filters[price][$gte]
filters[price][$lte]
filters[amount][$gt]
filters[manufacturer][$in]
filters[country][$in]
filters[uvResistance][$in]
```

Связь с категорией тоже фильтруется на стороне Strapi:

```ts
filters[kategoriis][slug][$eq]
```

## Поля товара

`Производитель`, `Страна происхождения` и `УФ-стойкость` хранятся в отдельных полях Strapi:

```ts
manufacturer: string
country: string
uvResistance: "да" | "нет"
```

Эти поля добавлены в модель товара:

```text
back/src/api/tovar/content-types/tovar/schema.json
```

Фронт больше не фильтрует товары по `product.characteristics`. Выбранные значения добавляются в Strapi query через `$in`, поэтому фильтрация работает по всей категории, а не только по товарам, которые уже попали на текущую страницу ответа.

## Варианты фильтров

Варианты чекбоксов собираются отдельно через `buildCatalogFilterOptionsUrl`.

Запрос получает только поля фильтров:

```ts
fields[0]=manufacturer
fields[1]=country
fields[2]=uvResistance
```

Чтобы не зависеть от `pagination[pageSize]=100`, страница сначала загружает первую страницу опций, читает `meta.pagination.pageCount`, затем догружает остальные страницы и собирает уникальные значения.

## Множественный выбор

Фильтры `manufacturer`, `country` и `uvResistance` поддерживают множественный выбор через чекбоксы. В URL это хранится повторением одного параметра:

```text
?manufacturer=A&manufacturer=B&country=Россия&country=Китай
```

Товар проходит фильтр, если его поле совпало хотя бы с одним выбранным значением внутри группы.

Между разными группами используется логика `AND`: товар должен подходить под выбранного производителя, страну и УФ-стойкость одновременно.

## Интерфейс

`CatalogFilters` является клиентским компонентом, потому что управляет открытием выпадающих окон фильтров.

Поведение:

- фильтры отображаются как компактные кнопки;
- внутри выпадающих окон используются чекбоксы;
- при открытии одного фильтра остальные открытые фильтры закрываются;
- кнопка `Применить` отправляет форму методом `GET`;
- активные фильтры подсвечиваются;
- `Сбросить` очищает фильтры, но сохраняет выбранную подкатегорию.

## Пагинация товаров

Список товаров по-прежнему запрашивается с `pagination[pageSize]=100`. Это ограничивает количество карточек, которые показываются на странице, но не ограничивает саму фильтрацию: Strapi сначала применяет `filters[...]`, а уже потом возвращает страницу результата.
